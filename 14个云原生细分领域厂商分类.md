

# [云原生细分领域厂商分类](https://dwiki.daocloud.io/pages/viewpage.action?pageId=113763589)

## 一、Obeservability

#### 开源：

- [Grafana Labs](https://grafana.com/)

是流行的开源分析和交互式可视化Web层。它支持时序数据的许多不同的存储后端。它可以连接到Graphite，InfluxDB，ElasticSearch，Prometheus等数据源。对于跟踪，它支持 Jaeger、Tempo、X-Ray 和 Zipkin 数据源。Grafana提供插件，仪表板，警报和不同的用户级访问权限，用于治理作为可观察性工具。



- [SigNoz](https://signoz.io/?utm_source=blog&utm_medium=article)

[SigNoz](https://signoz.io/?utm_source=blog&utm_medium=article)是一个全栈开源APM和可观察性工具。它通过产品路线图中的日志管理捕获指标和跟踪。SigNoz的优点之一是它是开源的，也可以托管在基础设施中。因此无需将数据发送给任何第三方。



#### 商业： 

- [New Relic ](https://newrelic.com/platform)

New Relic One是一个全面的基于云的可观察性平台，旨在帮助客户创建更完美的软件。它包括：

[遥测数据平台](https://docs.google.com/document/d/18x7XKRvrM0JgDf4pzpWLxiY7sgM2UmLcNhvGkeAMRjc/edit#bookmark=id.xpqxzg1sk60l)：使用托管、开放和统一的遥测数据库，从任何源收集、浏览和警报所有指标、事件、日志和跟踪。

[全栈可观察性：在](https://docs.google.com/document/d/18x7XKRvrM0JgDf4pzpWLxiY7sgM2UmLcNhvGkeAMRjc/edit#bookmark=id.4x4mjy4r7ki)一次互联体验中可视化、分析和排除整个软件堆栈故障。停止在不同的工具之间切换，疯狂地尝试将数据拼接在一起以找出出了什么问题。

[应用智能](https://docs.google.com/document/d/18x7XKRvrM0JgDf4pzpWLxiY7sgM2UmLcNhvGkeAMRjc/edit#bookmark=id.x2wnvonb1t2h)：利用 AIOps 功能更快地检测、理解和解决事件。



- [ScienceLogic](https://sciencelogic.com/)

通过数据驱动的自动化帮助 IT 运营领导者交付更好的业务成果。借助 ScienceLogic 和 AIOps，客户可以快速、大规模、实时地管理 IT 环境。



- [Auvik](https://www.auvik.com/)

Auvik基于云的网络管理软件使用户能够深入了解他们所管理的网络，并自动执行复杂且耗时的网络任务。实时网络映射和库存意味着用户知道什么在哪里，即使用户移动也是如此。在网络设备上自动进行配置备份和还原意味着用户无需手动操作即可降低网络风险。Auvik TrafficInsights显示谁在网络上，他们正在使用哪些应用程序，以及他们的流量在哪里。



- [Dynatrace](https://www.dynatrace.com/)

Dynatrace是一个以AI为中心的软件智能平台。它包括针对企业扩展的 APM 功能，具有单个事务级监视功能以及基础架构监视、AIOps、数字业务分析和数字体验监视。企业使用 Dynatrace 平台进行内部协作，并毫不费力地为用户提供价值。



- [Amazon CloudWatch](https://aws.amazon.com/cn/cloudwatch/)

Amazon CloudWatch 是一项原生 AWS 监控和可观察性服务，专为开发运营工程师、开发人员、站点可靠性工程师 （SRE） 和 IT 经理而构建。CloudWatch 为用户提供数据和可操作的见解，以监控应用程序、响应系统范围的性能变化、优化资源利用率以及获得运营运行状况的统一视图。CloudWatch 以日志、指标和事件的形式收集监控和操作数据，为用户提供在 AWS 和本地服务器上运行的 AWS 资源、应用程序和服务的统一视图。CloudWatch 可用于检测环境中的异常行为、设置警报、并排可视化日志和指标、采取自动化操作、排查问题以及发现见解，以保持应用程序平稳运行。



- [AppDynamics](http://appdynamics/)

AppDynamics 提供对组织业务和运营绩效各个方面的实时访问，因此他们可以预测问题，自动解决问题，并做出更明智、更确定的业务决策。应用程序智能为用户提供有关其软件应用程序的性能、用户体验和业务影响的业务和运营见解。



- [SolarWinds AppOptics](https://www.solarwinds.com/appoptics)

基于 SaaS 的 APM 和基础架构工具，适用于混合和云原生 IT 环境。主要特点有自动检测的应用程序服务拓扑图、查明性能问题的根本原因、性能监控、全栈可视性、现代基础设施监控和AWS监控。



- [Datadog](https://www.datadoghq.com/)

Datadog是面向云时代开发人员，IT运营团队，安全工程师和业务用户的监控，安全和分析平台。Datadog的SaaS平台集成并自动化了基础设施监控，应用程序性能监控和日志管理，以提供客户整个技术堆栈的统一，实时可观察性。Datadog被各行各业各种规模的组织使用，以实现数字化转型和云迁移，推动开发，运营，安全和业务团队之间的协作，加快应用程序的上市时间，缩短解决问题的时间，保护应用程序和基础架构，了解用户行为并跟踪关键业务指标。



- [Splunk APM](https://www.splunk.com/en_us/observability/apm-application-performance-monitoring.html)

APM 对于在环境扩展时帮助提供应用和服务的可见性和上下文至关重要。Splunk APM为构建现代应用程序并对其进行故障排除的工程师提供了先进的解决方案;通过完全保真度跟踪分析提供跨复杂云和微服务环境的上下文，AI 指导故障排除和高基数，以更快地识别和解决问题，AlwaysOn 分析提供服务瓶颈的代码级可见性，以及 OpenTelemetry 标准化，可在您采用新技术和新流程时提供更大的灵活性和控制力。



## 二、Service Mesh

#### 开源：

- [Istio](https://istio.io/)

[Istio 是一个基于 Envoy 构建的可扩展开源服务网格](https://istio.io/)，允许团队连接、保护、控制和观察服务。Istio 于 2017 年开源，是 IBM 和 Google 之间的持续合作，Google 贡献了原始组件，Lyft [于 2017 年将 Envoy 捐赠给了 Cloud Native Computing Foundation](https://medium.com/better-programming/what-is-lyft-envoy-9dfb45470f19)。



- [Linkerd](https://linkerd.io/)

Linkerd是社区驱动的，100%Apache许可的开源，它也是一个云原生计算基金会（CNCF）孵化项目。自2016年以来，它也一直存在，因此维护人员有时间解决问题。

使用 Linkerd 服务网格，应用程序可以[为其 Kubernetes 部署](https://techbeacon.com/enterprise-it/hackers-guide-kubernetes-security)添加可靠性、可观察性和安全性功能



#### 商业：

- [Consul Connect](https://www.hashicorp.com/products/consul/multi-platform-service-mesh)

[Consul Connect](https://www.hashicorp.com/products/consul/multi-platform-service-mesh/)专注于路由和分段，通过应用程序级sidecar代理提供服务到服务网络功能。Consult Connect强调应用程序安全性，代理为应用程序提供相互传输层安全性（TLS）连接以进行授权和加密。它的独特之处在于有两个代理选项。Connect 提供了自己的内置层代理用于测试，但也支持 Envoy。Connect强调可观察性，提供与工具的集成，以监控来自边车代理（如Prometheus）的数据。



- [Kuma](https://github.com/Kong/kuma)

Kuma是一个基于Envoy构建的与平台无关的控制平面。Kuma 提供网络功能来保护、观察、路由和增强服务之间的连接。Kuma 除了支持虚拟机之外，还支持 Kubernetes。



- [Traefik Mesh](https://doc.traefik.io/traefik-mesh/)

Traefik Mesh是一种轻巧，更简单的服务网格，从头开始设计，简单明了，易于安装和易于使用。

Traefik Mesh 建立在 Traefik 之上，适合作为 Kubernetes 集群中的事实服务网格，支持最新的服务网格接口规范 （SMI）。此外，Traefik Mesh默认为选择加入，这意味着在决定将其添加到网格之前，现有服务不受影响。



- [Red Hat OpenShift](https://cloud.redhat.com/learn/topics/service-mesh)

[OpenShift](https://www.openshift.com/learn/topics/service-mesh) 是红帽旗下的容器管理平台，可帮助"连接、管理和观察基于微服务的应用程序"。OpenShift被描述为一个混合云企业Kubernetes平台，它预装了众多功能，并拥有大量的企业采用率。

OpenShift Service Mesh 建立在开源 Istio 之上，带来了 Istio 控制和数据平面功能。OpenShift 通过由两个开源工具提供支持的跟踪和可见性功能增强了 Istio。OpenShift [使用 Jaeger](https://www.jaegertracing.io/) 进行分布式跟踪，从而可以更好地跟踪服务之间如何处理请求。

OpenShift [还使用 Kiali](https://kiali.io/) 为微服务配置、流量监控和跟踪分析增加了可观察性。



- [IBM Cloud Managed Istio](https://www.ibm.com/cloud/istio)

Managed Istio 作为 IBM Cloud™ Kubernetes Service 的一部分提供。该服务提供 Istio 的无缝安装、控制平面组件的自动更新和生命周期管理，以及与平台日志记录和监控工具的集成。将托管 Istio 集成添加到新的或现有的集群中，以完全控制您的微服务。



- [AWS App Mesh](https://aws.amazon.com/cn/app-mesh/?aws-app-mesh-blogs.sort-by=item.additionalFields.createdDate&aws-app-mesh-blogs.sort-order=desc&whats-new-cards.sort-by=item.additionalFields.postDateTime&whats-new-cards.sort-order=desc)

AWS App Mesh 是一种服务网格，它提供应用程序级联网，使服务能够跨多种类型的计算基础设施轻松相互通信。App Mesh 旨在为应用程序提供端到端的可见性和高可用性。使用 AWS App Mesh 不会产生额外费用。用户为与容器一起部署的代理消耗的 AWS 资源（EC2 实例或请求的 Fargate CPU 和内存）付费。



- [Kong Konnect](https://konghq.com/kong-konnect)

Konnect 在构建时充分考虑了最终用户的需求，以简化每个开发人员、架构师和操作员的独特工作流程。它包括Kong Gateway（也通过其开源社区版提供）和Kong Mesh，一种企业级服务网格，用于Kubernetes和VM上的多云和多集群。



## 三、CI/CD

#### 开源：

- [Jenkins](https://jenkins.io/)

Jenkins 是一个开源自动化服务器，其中进行集中构建和持续集成过程。它是一个基于Java的独立程序，带有适用于Windows，macOS和其他类Unix操作系统的软件包。有了数百个可用的插件，Jenkins 支持软件开发项目的构建、部署和自动化。



- [GoCD](https://www.gocd.org/)

来自ThoughtWorks的GoCD是一个开源工具，用于构建和发布支持CI / CD上现代基础架构的软件。



- [Spinnaker](https://www.spinnaker.io/) 

Spinnaker 是一个多云持续交付平台，支持跨不同的云提供商发布和部署软件更改，包括 AWS EC2、Kubernetes、Google Compute Engine、Google Kubernetes Engine、Google App Engine 等。



#### 商业：

- [GitLab](https://about.gitlab.com/)

GitLab是一套用于管理软件开发生命周期不同方面的工具。核心产品是基于Web的Git存储库管理器，具有问题跟踪，分析和Wiki等功能。

GitLab 允许您在每次提交或推送时触发生成、运行测试和部署代码。可以在虚拟机、Docker 容器或其他服务器上生成作业。



- [CircleCI](https://circleci.com/)

CircleCI 是一个 CI/CD 工具，支持快速软件开发和发布。CircleCI 允许跨用户管道实现自动化，从代码构建、测试到部署。

您可以将 CircleCI 与 GitHub、GitHub Enterprise 和 Bitbucket 集成，以便在提交新代码行时创建构建版本。CircleCI 还在云托管选项下托管持续集成，或者在私有基础设施上的防火墙后面运行。



- [TeamCity](https://www.jetbrains.com/teamcity/)

TeamCity 是 JetBrains 的构建管理和持续集成服务器。TeamCity 是一个持续集成工具，可帮助构建和部署不同类型的项目。TeamCity在Java环境中运行，并与Visual Studio和IDE集成。该工具可以安装在Windows和Linux服务器上，支持.NET和开放堆栈项目。



- [Bamboo](https://www.atlassian.com/software/bamboo)

Bamboo是一个持续集成服务器，可以自动管理软件应用程序版本，从而创建一个持续交付管道。涵盖了构建和功能测试、分配版本、标记版本、在生产环境中部署和激活新版本。



- [Buddy](https://buddy.works/)

Buddy 是一款 CI/CD 软件，它使用来自 GitHub、Bitbucket 和 GitLab 的代码构建、测试、部署网站和应用程序。它采用具有预安装语言和框架的 Docker 容器进行构建，以及 DevOps、监视和通知操作。



- [Codeship](https://codeship.com/) 

Codeship是一个托管平台，可多次支持早期和自动软件发布。它通过优化测试和发布流程，帮助软件公司更快地开发更好的产品。



- [Wercker](https://app.wercker.com/)

对于正在与 Docker 合作或正在考虑启动基于 Docker 的新项目的开发人员来说，Wercker 可能是一个合适的选择。Wercker 支持组织及其开发团队使用 CI/CD、微服务和 Docker。



- [Semaphore](https://semaphoreci.com/)

Semaphore是用于测试和部署软件项目的托管 CI/CD 服务。信号量通过基于拉取请求的开发过程建立 CI/CD 标准。



- [Nevercode](https://nevercode.io/)

Nevercode 支持移动应用的 CI/CD。它有助于更快地构建、测试和发布本机和跨平台应用。
